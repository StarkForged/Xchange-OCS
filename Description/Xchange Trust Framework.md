Xchange Trust Framework
Overview

The Xchange Trust Framework is a comprehensive reputation and credibility system designed to build trust between buyers and sellers in a peer-to-peer classified marketplace.

Unlike traditional marketplaces that rely solely on ratings or reviews, Xchange evaluates users using five independent trust pillars. These pillars collectively measure a user's identity, transaction history, reputation, marketplace behaviour, and compliance with platform policies.

The objective of this framework is to:

Help users identify trustworthy members.
Reduce fraud and scam attempts.
Encourage responsible marketplace behaviour.
Reward genuine participation.
Prevent manipulation through fake reviews, collusive transactions, or spam.
Allow honest users to gradually build and recover trust while taking strict action against repeated offenders.
Core Principle

Trust is earned, not purchased or automatically granted.

Every user begins by proving their identity. Over time, trust is built through successful transactions, positive community feedback, healthy marketplace activity, and adherence to marketplace policies.

The Trust Framework ensures that one excellent review does not make someone highly trusted, while one false accusation does not immediately destroy a user's reputation.

Only verified actions and administrator-confirmed violations affect long-term trust.

Pillar 1 — Identity & Verification (20 Points)

This pillar establishes whether a user is genuine and verifiable.

Identity verification forms the foundation of trust.

Verification	Points
Verified Email	+6
Verified Phone Number (OTP)	+6
Complete Profile	+6
Admin Verified Seller	+2
Maximum	20

A normal verified user reaches 18 points.

The final 2 points are reserved for administrator verification, making the Verified Seller badge meaningful instead of automatic.

Pillar 2 — Transaction History (20 Points)

This pillar rewards successful exchanges.

Rather than rewarding unlimited transactions, Xchange uses diminishing returns so quality matters more than quantity.

Completed Deals	Points Awarded
First Deal	+5
2nd–3rd Deal	+3 each
4th–6th Deal	+2 each
7th–10th Deal	+1 each
Above 10	No additional points
Maximum	20

This prevents experienced users from becoming unbeatable simply because they have completed hundreds of transactions.

Anti-Collusion

Transactions suspected to be artificially generated between related accounts are excluded from trust calculations until reviewed.

Examples include:

Strongly matching verified identity information
Repeated reciprocal trading patterns
Other suspicious behavioural signals

This prevents users from farming trust through alternate accounts.

Pillar 3 — Reviews & Ratings (20 Points)

Ratings alone are easy to manipulate.

Instead of using a simple average, Xchange calculates reputation using a Bayesian Weighted Rating.

Weighted Rating

=
(Average Rating × Number of Reviews
+
3.5 × 5)

÷

(Number of Reviews + 5)

This assumes every new user starts with five neutral reviews, preventing a single 5-star rating from creating an unrealistically high reputation.

Review Points
Component	Maximum
Weighted Rating	16
Genuine Review Volume Bonus	4
Total	20
Anti-Review Farming

Only the first two reviews exchanged between the same buyer and seller influence trust.

Additional reviews remain publicly visible but no longer affect the Trust Framework.

This prevents two users from repeatedly reviewing each other to increase reputation.

Advantages of the Xchange Trust Framework
Multi-dimensional trust evaluation instead of relying solely on ratings.
Strong protection against review farming and collusive transactions.
Progressive trust disclosure prevents unfair judgement of new users.
Only administrator-confirmed violations affect trust.
Transparent trust history and five-pillar breakdown improve explainability.
Recovery mechanisms reward sustained positive behaviour while preserving accountability for serious misconduct.
Trust is communicated through badges, reputation tiers, and meaningful indicators rather than a single number.
